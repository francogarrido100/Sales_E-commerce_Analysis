# E-commerce Sales & Inventory Analysis

An end-to-end data analytics project focused on cleaning, processing, and optimizing e-commerce sales and inventory data to drive strategic business decisions.

## 📌 Business Problem & Context
In fast-paced e-commerce operations, raw reports generated by platforms like Amazon often come with inconsistencies, missing values, and redundant tracking metrics. This project addresses the challenge of transforming fragmented, unstructured data into a clean, unified source of truth ready for business intelligence (BI) stakeholders.

## 📊 Data Source
The analysis is driven by retail operations data, primarily utilizing the **`Sale Report.csv`** dataset, which contains structured operational details:
* **Product Identification:** `SKU Code`, `Design No.`
* **Inventory Control:** `Stock` levels.
* **Product Attributes:** `Category`, `Size`, and `Color`.

## 🛠️ Project Roadmap & Architecture

### Phase 1: Pipeline Automation (`analysis.py`)
* Developed an automated lightweight Python workflow to programmatically ingest raw platform data.
* Handled initial schema filtering by isolating key operational dimensions and exporting a standardized, business-ready deliverable (`Sales_Report_Clean.xlsx`).

### Phase 2: Deep Data Cleaning & Standardization (`practica.ipynb`)
* Utilized **Pandas** and **Jupyter Notebooks** to perform advanced data quality checks.
* Standardized categorical variables (e.g., transforming inconsistent `Color` entries into uniform casing).
* Implemented logical imputation for missing data, converting `NaN` entries into explicit `UNKNOWN` flags to ensure robust calculation metrics in downstream reporting.

### Phase 3: Interactive Dashboard (Next Step)
* Connect the standardized datasets into **Looker Studio** to build an interactive dashboard tracking key performance indicators (KPIs), sizing demand, and stock optimization.
